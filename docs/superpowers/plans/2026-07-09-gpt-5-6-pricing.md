# GPT-5.6 Pricing Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add exact, effective-dated API USD and Codex credit pricing for GPT-5.6 Sol, Terra, and Luna, then prepare extension version `0.1.32` for a dual-platform release.

**Architecture:** Keep parsing, caching, aggregation, and rendering unchanged. Extend the checked-in schedules in `pricing.py`, prove the new behavior through unit and end-to-end tests, and preserve exact-model matching so unpublished variants remain visible but unpriced.

**Tech Stack:** Python 3.13, pytest, `uv`, TypeScript 5.7, Node.js 24, npm, VS Code VSIX packaging, GitHub Actions.

## Global Constraints

- Support only the exact model ids `gpt-5.6-sol`, `gpt-5.6-terra`, and `gpt-5.6-luna`; add no aliases or prefix matching.
- Apply both API USD and Codex credit rates from `2026-06-26T00:00:00Z`.
- Set `PRICING_AS_OF` to `2026-07-09`.
- Keep reasoning effort, including `ultra`, outside the pricing key and model chart grouping.
- Keep explicit API cache-write uplift unpriced because local Codex logs expose no distinct cache-write token field.
- Add no runtime dependency, live pricing request, parser schema, cache schema, chart, or VS Code command change.
- Bump the Python package, VS Code extension, and lockfile metadata to `0.1.32`.
- Do not create or push `v0.1.32` during implementation; release tagging requires a separate approval and triggers both platform builds and Marketplace publishing.

## File Map

- `src/codex_usage/pricing.py`: owns the checked-in effective-dated API and credit schedules.
- `tests/test_pricing.py`: proves exact rates, calculations, effective dates, and unknown-variant behavior.
- `tests/test_parser_aggregation.py`: proves a real `gpt-5.6-sol` plus `ultra` event flows through parsing and aggregation correctly.
- `tests/test_reporting_html.py`: keeps the partial-pricing warning covered with an actually unknown future variant.
- `pyproject.toml` and `uv.lock`: own Python package version metadata.
- `extensions/vscode/package.json` and `extensions/vscode/package-lock.json`: own Marketplace package version metadata.
- `CHANGELOG.md` and `extensions/vscode/CHANGELOG.md`: record repository and Marketplace-facing release notes.
- `README.md` and `extensions/vscode/README.md`: explain supported GPT-5.6 tiers and the cache-write limitation.
