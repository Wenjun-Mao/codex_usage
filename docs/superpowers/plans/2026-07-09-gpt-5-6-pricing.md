# GPT-5.6 Pricing Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add exact, effective-dated API USD and Codex credit pricing for GPT-5.6 Sol, Terra, and Luna, add the official `gpt-5.6` Sol alias, apply GPT-5.6 request-level long-context API pricing, and keep extension version `0.1.32` ready for release.

**Architecture:** Keep parsing, caching, aggregation, and rendering unchanged. Extend the checked-in schedules in `pricing.py`, model long-context API behavior as a reusable request-pricing contract on effective model-rate rows, prove the behavior through unit and end-to-end tests, and preserve exact-model matching so unpublished variants remain visible but unpriced.

**Tech Stack:** Python 3.13, pytest, `uv`, TypeScript 5.7, Node.js 24, npm, VS Code VSIX packaging, GitHub Actions.

## Global Constraints

- Support the exact model ids `gpt-5.6-sol`, `gpt-5.6-terra`, and `gpt-5.6-luna`.
- Add the official `gpt-5.6` alias to Sol API and Codex credit rows only.
- Keep exact model or explicit-alias matching. Do not add prefix or substring matching; `gpt-5.6-pro`, `gpt-5.6-mini`, and wrapper names remain unpriced.
- Apply GPT-5.6 API USD rates from `2026-06-26T00:00:00Z`.
- Apply GPT-5.6 Codex credit rates from `2026-07-09T00:00:00Z`.
- Set `PRICING_AS_OF` to `2026-07-09` and keep package/extension version `0.1.32`.
- Apply GPT-5.6 long-context API pricing only when one retained usage event has `usage.input_tokens > 272_000`. Exactly 272,000 input tokens remains short-context pricing; 272,001 input tokens uses long-context pricing.
- Long-context API pricing affects the full retained event: uncached input and cached input rates are 2x, and output rates are 1.5x. The long API rates are Sol $10/$1/$45, Terra $5/$0.50/$22.50, and Luna $2/$0.20/$9 per 1M uncached input, cached input, and output tokens.
- Do not apply the long-context multiplier to Codex credits. GPT-5.6 long-context pricing is API-only; Codex credit rates remain flat input, cached input, and output rates from the public rate card.
- Use the existing per-event positive token delta as request usage. A local audit found retained positive deltas matched request-level `last_token_usage`; cumulative session totals cannot trigger long-context pricing.
- Keep reasoning effort, including `ultra`, outside the pricing key and model chart grouping.
- Keep explicit API cache-write uplift unpriced because local Codex logs expose no distinct cache-write token field.
- Add no runtime dependency, live pricing request, parser schema, cache schema, chart, or VS Code command change.
- Do not create or push `v0.1.32` during implementation; release tagging requires a separate approval and triggers both platform builds and Marketplace publishing.

Add named request-pricing contracts associated with effective rate entries:

```python
@dataclass(frozen=True)
class RequestLevelLongContextPricing:
    input_token_threshold: int
    input_rate_multiplier: float
    cached_input_rate_multiplier: float
    output_rate_multiplier: float

    def applies_to(self, usage: TokenUsage) -> bool:
        return usage.input_tokens > self.input_token_threshold

    def apply(self, rate: ModelRate) -> ModelRate:
        return ModelRate(
            input_per_1m=rate.input_per_1m * self.input_rate_multiplier,
            cached_input_per_1m=rate.cached_input_per_1m * self.cached_input_rate_multiplier,
            output_per_1m=rate.output_per_1m * self.output_rate_multiplier,
        )


@dataclass(frozen=True)
class RequestPricingContract:
    long_context_pricing: RequestLevelLongContextPricing | None = None

    def rate_for_usage(self, base_rate: ModelRate, usage: TokenUsage) -> ModelRate:
        if self.long_context_pricing is not None and self.long_context_pricing.applies_to(usage):
            return self.long_context_pricing.apply(base_rate)
        return base_rate


STANDARD_REQUEST_PRICING = RequestPricingContract()
GPT_5_6_API_LONG_CONTEXT_PRICING = RequestPricingContract(
    long_context_pricing=RequestLevelLongContextPricing(
        input_token_threshold=272_000,
        input_rate_multiplier=2.0,
        cached_input_rate_multiplier=2.0,
        output_rate_multiplier=1.5,
    )
)
```

Extract schedule-entry lookup so `rate_for_model` can keep returning the base rate while `estimate_cost` can apply the request-pricing contract:

```python
def _rate_for_model(
    schedule: tuple[EffectiveModelRate, ...],
    model: str,
    at: datetime | None = None,
) -> ModelRate | None:
    entry = _schedule_entry_for_model(schedule, model, at)
    if entry is None:
        return None
    return entry.rate


def _schedule_entry_for_model(
    schedule: tuple[EffectiveModelRate, ...],
    model: str,
    at: datetime | None = None,
) -> EffectiveModelRate | None:
    normalized = _normalize_model_id(model)
    effective_at = _normalize_effective_at(at)
    candidates = [
        entry
        for entry in schedule
        if _matches_model(entry, normalized)
        and (effective_at is None or _normalize_effective_at(entry.effective_from) <= effective_at)
    ]
    if candidates:
        return max(
            candidates,
            key=lambda entry: _normalize_effective_at(entry.effective_from) or BASELINE_EFFECTIVE_FROM,
        )
    return None
```

Update API USD cost estimation to select the effective entry first and then apply request-level pricing to the retained event:

```python
def estimate_cost(usage: TokenUsage, model: str, at: datetime | None = None) -> CostBreakdown | None:
    entry = _schedule_entry_for_model(API_PRICING_USD_SCHEDULE, model, at)
    if entry is None:
        return None
    rate = entry.rate_for_usage(usage)

    uncached_input_usd = usage.uncached_input_tokens / 1_000_000 * rate.input_per_1m
    cached_input_usd = usage.cached_input_tokens / 1_000_000 * rate.cached_input_per_1m
    output_usd = usage.output_tokens / 1_000_000 * rate.output_per_1m
    return CostBreakdown(
        uncached_input_usd=uncached_input_usd,
        cached_input_usd=cached_input_usd,
        output_usd=output_usd,
        total_usd=uncached_input_usd + cached_input_usd + output_usd,
    )
```
