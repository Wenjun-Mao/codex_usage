# ADR 0002: Native HTML And SVG Dashboard

Status: Accepted

Date: 2026-06-16

## Context

The dashboard needed to work as a standalone local report and inside a VS Code webview. Adding a heavy charting stack would increase dependencies and complicate packaging.

## Decision

Render the report as self-contained HTML, CSS, and inline SVG generated by Python. Keep the webview script-free and compatible with strict CSP.

## Alternatives Considered

- Plotly or another JavaScript charting library. Richer interactions, but heavier assets and more CSP/webview work.
- Pandas plus image charts. Easy data work, but heavier dependencies and less themeable output.
- TypeScript-rendered charts. Better webview interactivity, but duplicates report logic and weakens CLI parity.

## Consequences

The report is portable, dependency-light, and easy to embed in VS Code. Interactions are limited, so hover behavior and chart polish must be implemented with careful HTML/SVG/CSS.

## Guardrails

No remote assets. No webview scripts unless a future ADR changes the security model.
